## 📌 项目简介

这是一个专为法律实践与监督设计的、开源的、AI驱动的司法与行政文书审查规则集（Skill）。它旨在系统化、自动化地检测判决书、仲裁裁决书、行政决定书中可能存在的**程序操作异常、证据采信双标、事实认定偏差、法律适用错误、自由裁量权滥用及逻辑闭环断裂**等问题。

该项目基于长期的法律实践观察、数百份裁判文书分析与对通用人工智能技术的应用总结而成。它不是要替代法官或律师的判断，而是作为一个 **“标准化、低成本、高效率”的初筛工具**，帮助普通当事人、法律从业者乃至案件质量评查人员，快速发现文书中可能被忽略的“异常点”。

## ✨ 核心功能

-   **15维深度扫描**：从程序正当性、证据一致性、事实认定（含26项精细化检测）、法律适用、类案偏离等15个维度进行全面审查。
-   **事实认定错误快速对照**：提供“翻案速查表”，将26种常见事实认定错误清晰列出，帮助用户像查字典一样快速定位问题。
-   **多重逻辑验证**：通过时间一致性检测、语义漂移分析、缺失信息（负空间）检测，发现文书中的“不言之隐”。
-   **量化与图分析**：进行类案偏离量化评分（Deviation Score），并构建程序行为链图（Procedure Graph），直观展示异常路径。
-   **对抗性校验**：内置“魔鬼代言人”反向审查机制，区分“主观偏袒”与“业务能力不足”，避免误判。
-   **多模型兼容**：可无缝接入当前主流通用大模型（如DeepSeek， GLM-4， Qwen-Max， Gemini等），即插即用。

## 🎯 适用场景

-   **普通当事人**：收到不公判决/裁决后，快速评估是否存在“硬伤”，为上诉或再审提供思路。
-   **律师/法务**：在代理案件或进行案件质量复核时，作为辅助工具，快速定位对方或己方文书的潜在弱点。
-   **案件评查员/监督机构**：从海量文书中高效筛查“高度异常”或“结构性偏差”的案件，实现精准监督。
-   **法学研究者**：批量分析裁判文书，研究不同维度异常的地域性、类型化分布特征。

## 🚀 快速开始

### 准备材料（以检测一份判决书为例）
1.  **必需**：待检测的判决书/裁决书全文。
2.  **强烈推荐**：起诉状/答辩状（用于对比争议焦点偏移）。
3.  **强烈推荐**：双方证据清单（用于检测证据采信一致性）。

### 使用方法

**方法一：使用AI编程助手/IDE**
1.  将本项目中的 `SKILL.md` 文件内容完整复制。
2.  打开支持Skill功能的AI IDE或命令行工具（如Trae CN， Cursor， Qwen CLI， Claude Code等）。
3.  新建对话，粘贴 `SKILL.md` 的内容作为系统指令。
4.  按照提示，上传您的文书材料，AI将自动执行分析并生成结构化报告。

**方法二：使用通用大模型Web端**
1.  复制 `SKILL.md` 文件内容。
2.  打开任意主流大模型对话界面（如DeepSeek， Kimi， GLM， Qwen， Gemini等）。
3.  将 `SKILL.md` 作为第一条消息发送，指令为：“请按照这个Skill的规则，帮我分析以下司法文书。”
4.  随后上传或粘贴您的文书材料。

## ⚠️ 重要声明

1.  **非法律意见**：本工具生成的所有报告均为基于规则的初步筛查结果，**不构成正式的法律意见**。任何重要决策（如是否上诉、申请再审）请务必咨询执业律师。
2.  **客观定位**：本工具旨在识别“异常点”而非直接定性“枉法裁判”。它区分技术性错误（如逻辑疏漏、事实误判）与结构性偏差，为后续的专业评估提供客观依据。
3.  **数据隐私**：所有分析均在本地或您所使用的大模型服务端进行。出于隐私考虑，建议在使用时对当事人姓名、具体案号等敏感信息进行匿名化处理。本项目开发者不收集任何用户数据。
